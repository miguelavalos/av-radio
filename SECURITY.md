# Security Policy

## Reporting A Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

If you believe you have found a vulnerability in this repository, report it privately to:

- `info@avalsys.com`

Include:

- a short description of the issue
- affected files or features
- reproduction steps if available
- impact assessment if known

We will review the report and respond privately.

## Private Config

This is a public open-source repository. Do not commit production keys, local config,
signing material, or private backend URLs. Login, Pro, and hosted backend values must
come from Infisical-generated local files. See
[`docs/private-config-and-infisical.md`](docs/private-config-and-infisical.md).
